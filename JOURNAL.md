### What was built — the shell and the spine only

- **The skeleton**, mirroring sera's app repos: pnpm workspace · TypeScript
  strict · vitest · Playwright · `scripts/run-gates.sh` · CI on push. The
  ops-console is vanilla TS + Vite, **zero framework** (mirrors sera's
  `apps/dispatch-console`).
- **The eight desks as hash ROUTES** (`src/desks/*.ts`, one file per desk,
  canon order Desk 1→8), each with the honest empty state « Aucune donnée — cet
  établi n'est pas encore branché. » **No desk is wired to real data; no command
  exists.** The law heads every desk file + README verbatim.
- **The maker-checker primitive** (`src/maker-checker.ts`) — same-identity is
  UNREPRESENTABLE: a type-level impossibility (`DistinctChecker` collapses the
  checker parameter to `never` when checker == maker) **and** a runtime refusal
  (string-compared ids throw `MakerCheckerSameIdentityError`). Both proven.
- **The audit-log primitive** (`src/audit-log.ts`) — append-only: who
  (`envelope.actor`) · what (`action`) · why (`reason`) · when
  (`envelope.serverTime`) · against which entity (`entity`) · with what evidence
  (`evidence`). Entries are deep-frozen and the store is a private `#field` with
  no remove/delete/update affordance. Mutation and deletion both refuse.
- **The 29-icon set** consumed byte-verbatim from canon (copied into
  `apps/ops-console/assets/icons/`; `src/icons.ts` generated by
  `scripts/gen-icons.mjs`); `test/icons.test.ts` proves each embedded glyph's
  sha256 == the canon manifest.
